# Обоснование тестового покрытия для `sec(x)`

## Классы эквивалентности

- **Корректные входные данные**
    - `x = 0`
    - малые `x` (положительные и отрицательные)
    - несколько значений внутри рабочей области `|x| <= pi/3`
    - валидные `eps > 0` разных порядков (`1e-14`, `1e-8`, `1e-3`)
- **Некорректные входные данные**
    - `eps = 0`, `eps < 0`, `eps = NaN`, `eps = Infinity`
    - `x = NaN`, `x = Infinity`
    - значения за пределами разрешенной области (в том числе `x = pi/2`)
    - `x` за пределами явно разрешенной области (`|x| > pi/3`)

## Проверяемые границы

- Граница рабочей области: значение `x` почти на границе `pi/3` (валидный случай).
- Точная граница рабочей области: `x = pi/3` (валидный случай).
- Значение сразу за границей `pi/3` (невалидный случай).
- Границы по `eps`: очень маленький, обычный и относительно большой.
- Значение `pi/2` как пример входа, отклоняемого по ограничению области.

## Почему покрытие достаточное

- Покрыты все основные ветви валидации входа.
- Проверена численная корректность на representative-наборе точек по всей разрешенной области.
- Проверена точность относительно эталона `1 / Math.cos(x)` для разных `eps`.
- Проверено математическое свойство четности: `sec(-x) = sec(x)`.
- Граничные и невалидные случаи отдельно тестируются, что снижает риск регрессий в условиях, где ряд применяется хуже всего.
